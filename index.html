유튜브 알고리즘 교정기 & 일본 영상 확장기
2026.03.10 – 2026.03.16  |  1인 개발 / AI 협업
프로젝트 개요
  • 유튜브 알고리즘이 한쪽으로 치우친 정보만 제공하는 확증편향 문제를 해결하기 위한 AI 기반 교정 도구
  • 사용자가 추천된 영상의 제목과 URL을 입력하면, AI가 성향을 분석하고 반대 관점의 검색어를 자동으로 제시하는 구조
  • API 접근이나 시청 기록 없이도 동작 가능한 비침투형 접근 방식 채택 — 영상 제목·URL만으로 분석
사용 기술 및 환경
  • AI Tools: ChatGPT API (GPT 기반 성향 분석 및 키워드 생성) — Gemini API 시도 후 키 오류로 전환
  • 개발 보조: Claude (코드 작성, 디버깅)
  • Environment: Python, PyQt5 (GUI), OpenAI API
주요 성과 및 결과
  • [워크플로우] URL, 영상 제목 입력 → AI 요약 및 키워드 추출 → 반대 성향 키워드 생성 → 유튜브 검색어 조합 제시의 5단계 파이프라인 구현
  • [정확도 검증] 동일 영상을 순서만 바꿔 제출해도 동일 성향이 출력됨을 확인, 재현성 입증
  • [트러블슈팅] Gemini API 키 활성화 지연 문제 발생 → ChatGPT API로 전환하여 즉시 해결
  • [자막 미지원 대응] 자막 없는 영상은 제목만으로 분석하는 fallback 로직 적용 — 정치 분야 특성상 제목에 성향이 명확히 드러남을 활용
  • [파생 버전] v1.1 일본 영상 확장기 — 음악 장르·분위기 분석 → 한국어 3개 + 일본어 3개 검색어 제시 구조로 변형
나의 역할 (기획 · 설계 · AI Orchestrator)
  • 문제 정의 및 아이디어 도출: 유튜브 알고리즘의 확증편향 구조를 분석하고, API·시청 기록 없이 동작 가능한 해결 방식을 독자적으로 고안
  • 입력 설계: 정확도와 사용 편의성 균형을 위해 최소 3개~최대 10개 입력 범위 설정, 재미 접근과 실사용 모두 포괄
  • UI 기획: 5페이지 흐름(입력 → 로딩 3단계 → 결과)을 PPT로 직접 설계 후 AI에게 구현 지시
  • 파생 버전 기획: 정치 교정기와 결이 다른 음악 확장기를 별도 파생으로 분리하고, 입력 범위를 1~3개로 재설정하여 독립 설계

v2 (진행중) — 정치 외 다른 분야로의 확장, 분야별 키워드 스위치 고민 중

AI 오케스트레이션: 교과 맞춤형 문제집 자동 생성 파이프라인
2026.02.22 – 2026.02.23  |  1인 개발 / AI 협업
프로젝트 개요
  • Claude와의 고도화된 프롬프트 엔지니어링을 통해 기획부터 로직 구현까지 진행한 AI 협업 프로젝트
  • 시중 문제집의 높은 비용과 수동 문서 작업의 번거로움을 해결하기 위해, AI가 생성한 문제를 즉시 PDF로 변환 및 출력하는 자동화 도구 개발
사용 기술 및 환경
  • AI Tools: GPT-4o mini (초등용), Gemini 2.5 Flash (교사 배포용) — API, 문제 생성
  • 개발 보조: Claude Sonnet (코드 작성, 디버깅)
  • Environment: Python, PyQt5 (GUI), OpenAI API 및 Gemini API, ReportLab (PDF 엔진)
주요 성과 및 결과
  • [워크플로우 통합] AI 문제 생성 → 전처리 → PDF 레이아웃 구성 → 인쇄 인터페이스의 완전 자동화
  • [트러블슈팅] 시스템 폰트 자동 탐색 로직을 제안하여 한글 렌더링 및 이식성 문제 해결
  • [사용자 경험] 다크모드 UI 및 실시간 미리보기 확대/축소 기능 도입, 생성된 문제·힌트·정답 직접 수정 기능 추가
  • [성능] 기본 3문제 생성 시 약 8초, 문제 추가당 약 1~2초 소요로 빠른 응답성 확보
  • [비용] 5문제 생성 시 GPT-4o mini API 평균 약 800토큰 소모 → 약 0.4원~0.5원 수준. 시중 문제집 5문제당 약 100~150원 대비 약 250배 저렴
나의 역할 (AI Orchestrator & Architect)
  • 요구사항 정의: 교육적 목적에 부합하는 문제 유형 및 난이도 조절 로직 설계, 인라인 문제 편집 및 확대 기능, 다크모드 UI 제안 및 디자인, 저장 방식 세분화 제안
  • 프롬프트 설계: JSON 구조 출력을 강제하고 교과 맞춤 페르소나를 주입하여 데이터 무결성 확보, 2단계 AI 교차검증 파이프라인 설계
  • 시스템 통합 및 디버깅: 모듈 간 데이터 흐름 조율, 32비트 Python 환경 호환 문제 트러블슈팅, bat 스크립트 기반 무설치 배포 방식 설계
  • 파생 버전 기획 및 배포: v1-edu를 교사 실사용 환경에 맞게 독립 브랜치로 설계하고 현직 교사에게 직접 전달

사용 기술 및 환경
  • Backend: PHP (Apache 서버 환경)
  • Frontend: HTML5, CSS3, JavaScript (반응형 UI/UX 구현)
  • Database: MySQL (관계형 DB 설계 및 데이터 무결성 확보)

  • [평가 체계] 중복 평가 제한 및 평균 점수 산출 방식을 통해 정보의 신뢰성 확보

3중 보안 여행 가방 안전장치 (Secure Travel Bag)
2025.09.05 – 2025.12.19  |  팀 프로젝트
프로젝트 개요
  • 유럽 등 해외 여행 시 소매치기 방지를 위한 3중 보안 시스템 설계 및 구현
  • 전도성 터치 감지, IR 센서, SOS 패닉 모드를 결합한 통합 보안 장치
사용 기술 및 환경
  • Hardware: IR Sensor, C1815 Transistor, LM3914, 555 Timer IC, 능동 부저
  • Logic Gates: 7404 (NOT Gate), 7408 (AND Gate)
  • Circuit Design: Proteus (회로 시뮬레이션)
  • Power: 9V / 5V 듀얼 전원
주요 구현 내용
  • Touch Loop: 맨손 접촉 시 전도성 감지를 통한 즉각 경보
  • IR Loop: 장갑/도구 사용 시도 감지 (비전도체 접근 차단)
  • SOS Mode: 555 타이머 기반 지속적 경보 시스템
  • LED 시각화: LM3914 활용, LED 바/도트 디스플레이로 위험 수준 표시
나의 역할
  • 디지털 회로 설계 및 논리 게이트 구성
  • 회로도 설계 및 시뮬레이션 검증
